# Backend Services

Эта папка содержит бэкенд-сервисы, которые работают отдельно от основного блокчейн-узла.

## Структура

```
backend/
├── api/          # REST API серверы, gRPC gateways
├── services/     # Микросервисы, фоновые сервисы
├── workers/      # Фоновые воркеры, задачи
├── utils/        # Утилиты для бэкенд-сервисов
└── README.md     # Этот файл
```

## Назначение

Backend сервисы - это дополнительные сервисы, которые:
- Работают отдельно от блокчейн-узла (`cmd/volnixd/`)
- Не являются частью основного приложения (`app/`)
- Предоставляют дополнительные API или функциональность

## Примеры сервисов

- **REST API сервер** - HTTP-обертка над gRPC для веб-приложений
- **Webhook сервисы** - обработка событий блокчейна
- **Индексирующие сервисы** - индексация данных блокчейна
- **Интеграции с внешними API** - интеграция с внешними сервисами
- **Фоновые воркеры** - обработка задач в фоне

## Правила организации

- Каждый сервис должен быть в своей подпапке
- Каждый сервис должен иметь `README.md` с описанием
- Конфигурационные файлы в корне сервиса
- Зависимости управляются через `go.mod` или `package.json` в корне сервиса
